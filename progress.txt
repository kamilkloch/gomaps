## Codebase Patterns

- **Server entry:** `server/src/index.ts` exports `app` (Express instance) for testability
- **Test framework:** Vitest in server/, tests in `server/tests/`
- **Client build:** `tsc -b && vite build` (project references)
- **Server build:** `tsc` (standard compilation)
- **Monorepo:** npm workspaces with `server` and `client` packages
- **Express version:** Express v5 (no need for @types/express body parsing)
- **DB layer:** `server/src/db/` — one file per entity, `schema.ts` for init, `types.ts` for interfaces, `index.ts` re-exports all
- **DB tests:** Use `createDatabase(tempPath)` for isolated test DBs; snake_case columns mapped to camelCase interfaces
